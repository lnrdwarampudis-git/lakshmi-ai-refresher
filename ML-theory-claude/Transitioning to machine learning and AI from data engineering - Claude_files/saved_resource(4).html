EXPLORATORY DATA ANALYSIS
EDA — Understanding Your Data Before Touching It
The step that determines whether your model succeeds or fails — before a single line of ML code
The 80/20 rule of ML: 80% of your time is data preparation. A perfectly tuned model on dirty data will be beaten every time by a simple model on clean data. EDA is not optional — it IS the work.
What EDA tells you
Data quality problems
· Missing values — how many, which pattern
· Duplicates — exact or near-duplicate rows
· Wrong dtypes — numbers stored as strings
· Inconsistent categories — "Male"/"male"/"M"
· Impossible values — age=-5, salary=9999999
· Date parsing errors
Distribution insights
· Skewness — long right/left tail
· Bimodal — two separate populations
· Heavy tails — outliers present
· Normal vs uniform distribution
· Class imbalance in target
· Feature ranges and scales
Relationships between features
· Highly correlated pairs — redundant
· Non-linear relationships
· Feature-target correlation strength
· Interaction effects between features
· Confounding variables
· Temporal patterns in time data
The EDA checklist — run in this order
1
Shape + dtypes + memory
df.shape, df.dtypes, df.info(), df.memory_usage(). Understand what you have before going deeper.
2
Missing values — count + pattern
df.isnull().sum(). Visualise missingness pattern. Is it random (MCAR), conditional (MAR), or systematic (MNAR)? The pattern determines the imputation strategy.
3
Duplicates
df.duplicated().sum(). Check exact duplicates and near-duplicates. Remove before any analysis.
4
Target variable analysis
Distribution of y. Class balance for classification. Skewness for regression. This determines your loss function and evaluation metric.
5
Univariate analysis — each feature individually
Histograms for numerical. Bar charts for categorical. Check range, skewness, outliers, cardinality.
6
Bivariate analysis — feature vs target
Scatter plots, box plots, correlation. Which features are most predictive? Any non-linear relationships?
7
Multivariate analysis — feature interactions
Correlation matrix, pair plots, VIF. Find multicollinearity. Find interaction features worth engineering.
8
Outlier detection
IQR method, Z-score, isolation forest. Decide: remove, cap, or transform?
9
Data quality fixes
Fix dtypes, standardise categories, handle impossible values, correct date formats.
10
Document findings → decide preprocessing strategy
Write down: which features need imputation, encoding, scaling, removal, transformation. This becomes your preprocessing pipeline.

DATA CLEANING — MISSING VALUES
Handling Missing Values — Every Strategy with Code
Understanding WHY data is missing determines HOW to handle it
Three types of missingness — must identify before choosing strategy:
MCAR (Missing Completely At Random) — no pattern, safe to impute or drop
MAR (Missing At Random) — missing depends on other observed features — use conditional imputation
MNAR (Missing Not At Random) — missing value is related to the missing value itself — most dangerous, needs domain knowledge
When to DROP
· Column missing > 40-50% — not enough signal
· Row missing majority of features
· MNAR with no fix possible
· Feature adds no predictive value anyway

Never drop without checking importance first
When to IMPUTE (simple)
· Missing < 5% — use mean/median/mode
· MCAR pattern — random, safe to fill
· Numerical: median (robust to outliers)
· Categorical: most frequent or "Unknown"
· Time series: forward fill (ffill)

Simplest — always try first
When to use ADVANCED imputation
· Missing 5-40% with patterns
· MAR — depends on other features
· KNN imputation — uses similar rows
· IterativeImputer (MICE) — multivariate
· Model-based — predict missing from others

Best accuracy but slower

Golden rule: Always add a missing indicator column BEFORE imputing. The fact that a value was missing is itself information the model can learn from. A customer with missing income is different from one with a recorded income — even after you fill it with the median.
DATA CLEANING — OUTLIERS
Outlier Detection and Treatment
Finding extreme values — decide to remove, cap, transform, or keep
Critical question before treating outliers: Is this a data error or a real extreme value? A salary of $10M may be an outlier but it is real. A salary of -$5000 is a data error. The treatment is completely different.
Detection methods
IQR method: outlier if < Q1-1.5×IQR or > Q3+1.5×IQR. Most common, non-parametric.

Z-score: outlier if |z| > 3. Assumes normal distribution. Sensitive to own outliers.

Modified Z-score: uses median instead of mean. Robust version of Z-score.

Isolation Forest: ML-based. Best for multivariate outliers. No distribution assumption.
Treatment strategies
Remove: data errors, impossible values (negative age). Use sparingly.

Cap/Winsorize: replace extreme values with percentile bounds (e.g. 1st–99th). Keeps row, reduces extreme influence.

Log transform: compresses right-skewed distributions. Handles multiplicative outliers.

Keep: if real and important (fraud detection — outliers ARE the signal!).
# ══════════════════════════════════════════════════════════ # OUTLIER DETECTION AND TREATMENT — All Methods # ══════════════════════════════════════════════════════════ import numpy as np import pandas as pd from scipy import stats from sklearn.ensemble import IsolationForest from sklearn.preprocessing import RobustScaler import warnings warnings.filterwarnings('ignore') np.random.seed(42) n = 400 df = pd.DataFrame({ 'age': np.append(np.random.normal(35, 8, n-10), [-5, 200, 150, 0, -10, 999, 120, -3, 180, 250]), 'salary': np.append(np.random.lognormal(10.8, 0.4, n-5), [0, -1000, 9999999, 500, -500]), 'score': np.append(np.random.normal(60, 15, n-8), [300, -50, 250, -30, 400, 500, -100, 350]), 'target': np.random.randint(0, 2, n) }) num_cols = ['age', 'salary', 'score'] # ── Method 1: IQR ───────────────────────────────────────── print("── METHOD 1: IQR Method ──") def detect_outliers_iqr(series, multiplier=1.5): Q1 = series.quantile(0.25) Q3 = series.quantile(0.75) IQR = Q3 - Q1 lower = Q1 - multiplier * IQR upper = Q3 + multiplier * IQR mask = (series < lower) | (series > upper) return mask, lower, upper for col in num_cols: mask, lo, hi = detect_outliers_iqr(df[col]) print(f" {col:8s}: {mask.sum():3d} outliers bounds=[{lo:.1f}, {hi:.1f}] " f"min={df[col].min():.1f} max={df[col].max():.1f}") # ── Method 2: Z-score ───────────────────────────────────── print("\n── METHOD 2: Z-score (|z|>3) ──") for col in num_cols: z = np.abs(stats.zscore(df[col].fillna(df[col].median()))) n_out = (z > 3).sum() print(f" {col:8s}: {n_out:3d} outliers with |z|>3") # ── Method 3: Modified Z-score (robust) ─────────────────── print("\n── METHOD 3: Modified Z-score (uses median) ──") def modified_zscore(series): median = series.median() mad = np.median(np.abs(series - median)) return 0.6745 * (series - median) / (mad + 1e-9) for col in num_cols: mz = np.abs(modified_zscore(df[col].fillna(df[col].median()))) n_out = (mz > 3.5).sum() print(f" {col:8s}: {n_out:3d} outliers with modified |z|>3.5") # ── Method 4: Isolation Forest (multivariate) ───────────── print("\n── METHOD 4: Isolation Forest (multivariate) ──") iso = IsolationForest(contamination=0.05, random_state=42, n_jobs=-1) X_iso = df[num_cols].fillna(df[num_cols].median()) outlier_pred = iso.fit_predict(X_iso) n_out_iso = (outlier_pred == -1).sum() print(f" Multivariate outliers detected: {n_out_iso}") print(f" ({n_out_iso/len(df)*100:.1f}% of data)") df['iso_outlier'] = (outlier_pred == -1).astype(int) # ── Treatment 1: Remove ─────────────────────────────────── print("\n── TREATMENT 1: Remove data errors ──") df_clean = df.copy() # Remove clearly impossible values (domain knowledge) df_clean = df_clean[df_clean['age'].between(0, 120)] df_clean = df_clean[df_clean['salary'] > 0] df_clean = df_clean[df_clean['score'].between(0, 100)] print(f"Removed {len(df)-len(df_clean)} impossible values") print(f"Rows: {len(df)} → {len(df_clean)}") # ── Treatment 2: Winsorize/Cap ──────────────────────────── print("\n── TREATMENT 2: Winsorize (cap at percentiles) ──") df_winsor = df.copy() for col in num_cols: lower = df_winsor[col].quantile(0.01) upper = df_winsor[col].quantile(0.99) before_range = df_winsor[col].max() - df_winsor[col].min() df_winsor[col] = df_winsor[col].clip(lower=lower, upper=upper) after_range = df_winsor[col].max() - df_winsor[col].min() print(f" {col:8s}: range {before_range:.1f} → {after_range:.1f}") # ── Treatment 3: Log transform ──────────────────────────── print("\n── TREATMENT 3: Log transform (for right-skewed) ──") df_log = df.copy() for col in ['salary']: skew_before = df_log[col].skew() df_log[col+'_log'] = np.log1p(df_log[col].clip(lower=0)) skew_after = df_log[col+'_log'].skew() print(f" {col}: skewness {skew_before:.2f} → {skew_after:.2f}") # ── Treatment 4: RobustScaler ───────────────────────────── print("\n── TREATMENT 4: RobustScaler (IQR-based scaling) ──") rs = RobustScaler() # centers at median, scales by IQR — outlier-resistant X_robust = rs.fit_transform(df[num_cols].fillna(0)) print(f" After RobustScaler: mean≈0, std varies by IQR") print(f" Age range: [{X_robust[:,0].min():.2f}, {X_robust[:,0].max():.2f}]") # ── Summary ─────────────────────────────────────────────── print("\n── OUTLIER TREATMENT DECISION GUIDE ──") guide = [ ('Data errors (impossible values)', 'Remove rows', 'age=-5, salary=-1000'), ('Real extremes, little ML impact', 'Keep + RobustScaler', 'High earners'), ('Right-skewed distribution', 'Log/sqrt transform', 'House prices, salary'), ('Real extremes, affect model', 'Winsorize at 1%-99%', 'Cap at percentiles'), ('Multivariate anomalies', 'Isolation Forest flag', 'Fraud detection'), ('Target is the outlier', 'Keep! Outlier = signal', 'Fraud, anomaly det.'), ] print(f"{'Situation':40s} {'Treatment':30s} {'Example':25s}") print("-"*100) for s, t, e in guide: print(f"{s:40s} {t:30s} {e}")
Never blindly remove outliers in fraud detection or anomaly detection. The outliers ARE the signal you are trying to find. Removing them defeats the purpose of the entire model. Always ask "is this outlier the thing I want to predict?" before removing anything.
DATA TRANSFORMATION
Encoding Categorical Features + Scaling Numerical Features
Making data digestible for ML algorithms — every technique explained
Why encoding matters
ML algorithms work with numbers. Categorical features like "red/blue/green" or "NYC/LA/Chicago" must be converted to numbers. Wrong encoding = model learns wrong relationships.

Nominal = no order (city, color) → one-hot
Ordinal = has order (low/mid/high) → label encode with correct order
Why scaling matters
Age ranges 0–100. Salary ranges 0–500,000. Without scaling, gradient descent updates salary weights far faster than age weights — the model is biased toward large-scale features.

Tree models — do NOT need scaling
Linear/SVM/KNN/NN — MUST scale
Fit on train only!
Always fit your encoder/scaler on training data only. Then use transform() on val and test. Fitting on all data leaks test distribution into training.

scaler.fit(X_train) ← correct
scaler.transform(X_val) ← correct
scaler.fit(X_all) ← DATA LEAKAGE!

Label Encoding (use only for ordinal features)") edu_order = ['High School', 'Bachelor', 'Master', 'PhD'] oe = OrdinalEncoder(categories=[edu_order]) edu_encoded = oe.fit_transform(X_train[['education']]) print(f" Education mapping: {dict(zip(edu_order, range(len(edu_order))))}") print(f" Sample: {X_train['education'][:5].tolist()}") print(f" Encoded: {edu_encoded[:5].ravel().tolist()}") # ── Encoding 2: One-Hot Encoding (nominal, low cardinality) ─ print("\n2. One-Hot Encoding (nominal, few categories)") ohe = OneHotEncoder(drop='first', sparse_output=False, handle_unknown='ignore') city_encoded = ohe.fit_transform(X_train[['city']]) city_cols = ohe.get_feature_names_out(['city']) print(f" Original cities: {X_train['city'].unique().tolist()}") print(f" Created columns: {city_cols.tolist()}") print(f" Sample rows:\n{pd.DataFrame(city_encoded[:3], columns=city_cols)}") # ── Encoding 3: Target Encoding (high cardinality) ──────── print("\n3. Target Encoding (for high-cardinality categorical)") # Replaces category with mean target value # IMPORTANT: compute on training data only to avoid leakage X_train_te = X_train.copy(); X_test_te = X_test.copy() city_target_mean = X_train_te.join(y_train).groupby('city')['target'].mean() X_train_te['city_target_enc'] = X_train_te['city'].map(city_target_mean) X_test_te['city_target_enc'] = X_test_te['city'].map(city_target_mean) # Handle unseen categories in test X_test_te['city_target_enc'].fillna(y_train.mean(), inplace=True) print(f" City target means: {city_target_mean.round(3).to_dict()}") # ── Encoding 4: Frequency Encoding ──────────────────────── print("\n4. Frequency Encoding (replace with category frequency)") city_freq = X_train['city'].value_counts(normalize=True) X_train['city_freq'] = X_train['city'].map(city_freq) X_test['city_freq'] = X_test['city'].map(city_freq).fillna(0) print(f" City frequencies: {city_freq.round(3).to_dict()}") # ── Encoding 5: Binary Encoding ─────────────────────────── print("\n5. Binary Encoding (gender — 2 categories)") gender_map = {'Male': 0, 'Female': 1} X_train['gender_bin'] = X_train['gender'].map(gender_map) X_test['gender_bin'] = X_test['gender'].map(gender_map) print(f" Gender: Male=0, Female=1") # ══════════════════════════════════════════════ # NUMERICAL SCALING # ══════════════════════════════════════════════ print("\n\nNUMERICAL SCALING") num_cols = ['age', 'salary', 'score'] X_num_tr = X_train[num_cols].values X_num_te = X_test[num_cols].values # ── Scaling 1: StandardScaler (Z-score) ─────────────────── print("\n1. StandardScaler (Z-score) — mean=0, std=1") ss = StandardScaler() X_std = ss.fit_transform(X_num_tr) print(f" Age: mean={ss.mean_[0]:.1f}, std={np.sqrt(ss.var_[0]):.1f}") print(f" After scaling — mean={X_std[:,0].mean():.3f}, std={X_std[:,0].std():.3f}") print(f" USE WHEN: linear models, SVM, neural networks") # ── Scaling 2: MinMaxScaler ─────────────────────────────── print("\n2. MinMaxScaler — scales to [0,1] range") mms = MinMaxScaler() X_mm = mms.fit_transform(X_num_tr) print(f" After scaling — min={X_mm.min():.3f}, max={X_mm.max():.3f}") print(f" USE WHEN: when 0-1 range needed (neural nets, KNN, image data)") print(f" AVOID WHEN: outliers present (they compress everything else)") # ── Scaling 3: RobustScaler ─────────────────────────────── print("\n3. RobustScaler — uses IQR, robust to outliers") rs = RobustScaler() X_rob = rs.fit_transform(X_num_tr) print(f" Salary center: {rs.center_[1]:.0f} scale: {rs.scale_[1]:.0f}") print(f" USE WHEN: outliers present (salary, house prices)") # ── Scaling 4: Log Transform ────────────────────────────── print("\n4. Log Transform — for right-skewed distributions") salary_skew_before = pd.Series(X_num_tr[:,1]).skew() X_log = np.log1p(X_num_tr) # log1p = log(1+x), handles 0 values salary_skew_after = pd.Series(X_log[:,1]).skew() print(f" Salary skewness: {salary_skew_before:.3f} → {salary_skew_after:.3f}") print(f" USE WHEN: exponential/power-law distributions (salary, price, count)") # ── Scaling 5: QuantileTransformer ─────────────────────── print("\n5. QuantileTransformer — maps to uniform/normal distribution") qt = QuantileTransformer(output_distribution='normal', random_state=42) X_qt = qt.fit_transform(X_num_tr) print(f" Maps any distribution to Gaussian — most powerful transformation") print(f" After: skewness={pd.Series(X_qt[:,1]).skew():.3f}") print(f" USE WHEN: extreme skew, non-normal distributions") # ── Scaling 6: PowerTransformer (Yeo-Johnson) ──────────── print("\n6. PowerTransformer (Yeo-Johnson)") pt = PowerTransformer(method='yeo-johnson') # handles negatives unlike Box-Cox X_pt = pt.fit_transform(X_num_tr) print(f" Salary skewness: {pd.Series(X_pt[:,1]).skew():.3f}") print(f" USE WHEN: need Gaussian output, handles negative values") # ══════════════════════════════════════════════ # COMBINED PIPELINE with ColumnTransformer # ══════════════════════════════════════════════ print("\n\n── PRODUCTION PIPELINE: ColumnTransformer ──") from sklearn.compose import ColumnTransformer numeric_features = ['age', 'salary', 'score'] ordinal_features = ['education'] nominal_features = ['city', 'gender'] preprocessor = ColumnTransformer([ ('num', StandardScaler(), numeric_features), ('ordinal', OrdinalEncoder(categories=[['High School','Bachelor','Master','PhD'], ['Male','Female']]), ['education', 'gender']), ('nominal', OneHotEncoder(drop='first', handle_unknown='ignore'), ['city']), ], remainder='drop') from sklearn.ensemble import RandomForestClassifier from sklearn.metrics import roc_auc_score full_pipe = Pipeline([ ('prep', preprocessor), ('clf', RandomForestClassifier(n_estimators=100, random_state=42)) ]) full_pipe.fit(X_train, y_train) print(f"Pipeline AUC: {roc_auc_score(y_test, full_pipe.predict_proba(X_test)[:,1]):.4f}") print("Feature names after transformation:") try: print(full_pipe['prep'].get_feature_names_out()) except: print("(get_feature_names_out available in sklearn >= 1.0)") # ── Comparison table ────────────────────────────────────── print("\n── SCALER COMPARISON GUIDE ──") scalers = [ ('StandardScaler', 'Z-score: (x-mean)/std', 'Linear, SVM, NN, LR', 'Outliers affect mean/std'), ('MinMaxScaler', 'Scale to [0,1]', 'KNN, NN, no outliers', 'Sensitive to outliers'), ('RobustScaler', '(x-median)/IQR', 'When outliers present', 'Less sensitive to outliers'), ('Log transform', 'log(1+x)', 'Right-skewed (salary)', 'Only for positive values'), ('QuantileTransf', 'Map to normal/uniform', 'Extreme skew', 'Changes distribution shape'), ('No scaling', 'Raw values', 'Tree-based models', 'Bad for gradient methods'), ] print(f"{'Scaler':20s} {'Formula':25s} {'Use when':30s} {'Limitation'}") for name,form,use,lim in scalers: print(f"{name:20s} {form:25s} {use:30s} {lim}")
Tree-based models (Random Forest, XGBoost, Decision Trees) do NOT need scaling — they split on thresholds so the scale is irrelevant. Linear models, SVM, KNN, and neural networks MUST be scaled — they are distance or gradient based and scale matters enormously.
FEATURE ENGINEERING
Creating New Features — The Biggest ROI in ML
Feature engineering beats algorithm selection every single time
Why feature engineering matters more than model choice: A logistic regression with excellent engineered features will beat XGBoost with raw features. The model can only learn what is in the data — if the signal is buried, no algorithm can find it. Your job is to surface the signal.
Mathematical transformations
· Log — compress right-skewed
· Square root — moderate skew
· Square / cube — amplify non-linearity
· Polynomial (x², x³, x₁×x₂) — capture curves
· Reciprocal (1/x) — diminishing returns
· Absolute value — magnitude only
Interaction features
· Multiply two features: x₁×x₂
· Ratio: x₁/x₂ (salary/age = earning rate)
· Difference: x₁-x₂ (current-past score)
· Bin+encode: age_group×income_group
· Polynomial combos: from PolynomialFeatures
· Domain-specific: BMI = weight/height²
Aggregation features
· GroupBy mean/max/min/std/count
· Rolling window stats (time series)
· Lag features — previous value
· Cumulative sum/mean
· Rank within group
· Frequency of category in group
# ══════════════════════════════════════════════════════════ # FEATURE ENGINEERING — All Techniques with Real Impact # ══════════════════════════════════════════════════════════ import numpy as np import pandas as pd from sklearn.preprocessing import PolynomialFeatures, KBinsDiscretizer from sklearn.datasets import fetch_california_housing, load_breast_cancer from sklearn.model_selection import cross_val_score, train_test_split from sklearn.ensemble import RandomForestRegressor, RandomForestClassifier from sklearn.preprocessing import StandardScaler from sklearn.pipeline import Pipeline import warnings warnings.filterwarnings('ignore') # ── Dataset ─────────────────────────────────────────────── housing = fetch_california_housing(as_frame=True) df = housing.frame.copy() X_raw = housing.data y = housing.target # Baseline score rf = RandomForestRegressor(n_estimators=100, random_state=42, n_jobs=-1) baseline_r2 = cross_val_score(rf, X_raw, y, cv=5, scoring='r2').mean() print(f"Baseline R² (raw features): {baseline_r2:.4f}") print(f"Features: {X_raw.shape[1]}") # ══════════════════════════════════════════════ # 1. MATHEMATICAL TRANSFORMATIONS # ══════════════════════════════════════════════ print("\n── 1. Mathematical Transformations ──") df_eng = df.copy() # Log transform skewed features skewed_cols = [] for col in df.drop('MedHouseVal',axis=1).columns: if abs(df[col].skew()) > 1: df_eng[col+'_log'] = np.log1p(df[col].clip(lower=0)) skewed_cols.append(col) print(f" log({col}): skew {df[col].skew():.2f} → {df_eng[col+'_log'].skew():.2f}") # Square root transform for col in ['Population', 'HouseAge']: df_eng[col+'_sqrt'] = np.sqrt(df[col].clip(lower=0)) print(f" sqrt({col}): skew {df[col].skew():.2f} → {df_eng[col+'_sqrt'].skew():.2f}") # ══════════════════════════════════════════════ # 2. INTERACTION FEATURES (domain knowledge) # ══════════════════════════════════════════════ print("\n── 2. Interaction Features ──") # Rooms per person — more meaningful than raw rooms df_eng['rooms_per_person'] = df['AveRooms'] / (df['Population'] + 1) df_eng['bedrooms_per_room'] = df['AveBedrms'] / (df['AveRooms'] + 1) df_eng['population_density'] = df['Population'] / (df['AveRooms'] + 1) df_eng['income_per_person'] = df['MedInc'] / (df['AveOccup'] + 1) df_eng['location_score'] = df['Latitude'] * df['Longitude'] df_eng['coast_distance'] = np.sqrt((df['Latitude']-34)**2 + (df['Longitude']+118)**2) new_feats = ['rooms_per_person','bedrooms_per_room','population_density', 'income_per_person','location_score','coast_distance'] for f in new_feats: print(f" {f:30s}: mean={df_eng[f].mean():.3f} std={df_eng[f].std():.3f}") # ══════════════════════════════════════════════ # 3. BINNING / DISCRETISATION # ══════════════════════════════════════════════ print("\n── 3. Binning / Discretisation ──") # Convert continuous to bins — captures non-linear effects kbd = KBinsDiscretizer(n_bins=5, encode='ordinal', strategy='quantile') df_eng['income_bin'] = kbd.fit_transform(df[['MedInc']]) df_eng['age_bin'] = KBinsDiscretizer(n_bins=4, encode='ordinal', strategy='uniform').fit_transform(df[['HouseAge']]) print(" Income bins (quantile-based):") for i, (lo, hi) in enumerate(zip(kbd.bin_edges_[0][:-1], kbd.bin_edges_[0][1:])): count = (df_eng['income_bin']==i).sum() print(f" Bin {i}: [{lo:.2f}, {hi:.2f}] n={count}") # Percentile bins — rank within distribution df_eng['income_percentile'] = df['MedInc'].rank(pct=True) print(f"\n Income percentile range: {df_eng['income_percentile'].min():.3f} — {df_eng['income_percentile'].max():.3f}") # ══════════════════════════════════════════════ # 4. POLYNOMIAL FEATURES # ══════════════════════════════════════════════ print("\n── 4. Polynomial Features ──") top_features = ['MedInc', 'HouseAge', 'AveRooms', 'Latitude'] poly = PolynomialFeatures(degree=2, interaction_only=False, include_bias=False) X_poly = poly.fit_transform(df[top_features]) poly_names = poly.get_feature_names_out(top_features) print(f" {len(top_features)} features → {X_poly.shape[1]} polynomial features") print(f" New features: {[n for n in poly_names if '^' in n or ' ' in n][:10]}") # ══════════════════════════════════════════════ # 5. DATE/TIME FEATURES # ══════════════════════════════════════════════ print("\n── 5. Date/Time Feature Extraction ──") # Example with synthetic dates dates = pd.date_range('2020-01-01', periods=100, freq='D') df_dates = pd.DataFrame({'date': dates, 'value': np.random.randn(100)}) df_dates['year'] = df_dates['date'].dt.year df_dates['month'] = df_dates['date'].dt.month df_dates['day'] = df_dates['date'].dt.day df_dates['dayofweek'] = df_dates['date'].dt.dayofweek # 0=Mon, 6=Sun df_dates['is_weekend'] = (df_dates['dayofweek'] >= 5).astype(int) df_dates['quarter'] = df_dates['date'].dt.quarter df_dates['week'] = df_dates['date'].dt.isocalendar().week.astype(int) df_dates['days_since'] = (df_dates['date'] - df_dates['date'].min()).dt.days # Cyclical encoding for month/day (preserves circular nature) df_dates['month_sin'] = np.sin(2*np.pi*df_dates['month']/12) df_dates['month_cos'] = np.cos(2*np.pi*df_dates['month']/12) df_dates['dow_sin'] = np.sin(2*np.pi*df_dates['dayofweek']/7) df_dates['dow_cos'] = np.cos(2*np.pi*df_dates['dayofweek']/7) print(f" Date features created: {[c for c in df_dates.columns if c not in ['date','value']]}") print(" Note: cyclical sin/cos ensures Dec→Jan continuity, not a jump from 12 to 1") # ══════════════════════════════════════════════ # 6. AGGREGATION FEATURES # ══════════════════════════════════════════════ print("\n── 6. Aggregation Features (GroupBy) ──") # Add income_bin as group df_agg = df.copy() df_agg['income_bin'] = pd.qcut(df['MedInc'], 5, labels=False) # Aggregate stats per income group grp_stats = df_agg.groupby('income_bin')['AveRooms'].agg(['mean','std','max']) grp_stats.columns = ['rooms_mean_by_income','rooms_std_by_income','rooms_max_by_income'] df_agg = df_agg.join(grp_stats, on='income_bin') for col in ['rooms_mean_by_income','rooms_std_by_income']: corr = df_agg[col].corr(y) print(f" {col}: corr with target = {corr:.3f}") # ══════════════════════════════════════════════ # 7. IMPACT OF FEATURE ENGINEERING # ══════════════════════════════════════════════ print("\n── 7. Impact Comparison ──") feature_sets = { 'Raw 8 features': X_raw, '+ log transforms': np.hstack([X_raw, np.log1p(df[skewed_cols].clip(0))]), '+ interaction features': np.hstack([X_raw, df_eng[new_feats].values]), '+ bins + interactions': np.hstack([X_raw, df_eng[new_feats+['income_bin','age_bin']].values]), '+ polynomial (top 4 feats)': np.hstack([X_raw, X_poly]), 'All engineered features': df_eng.drop('MedHouseVal',axis=1).fillna(0).values, } for name, X_fe in feature_sets.items(): r2 = cross_val_score(rf, X_fe, y, cv=5, scoring='r2', n_jobs=-1).mean() improvement = r2 - baseline_r2 print(f" {name:35s}: R²={r2:.4f} improvement={improvement:+.4f} features={X_fe.shape[1]}")
Most impactful feature engineering techniques in order of ROI: (1) Domain-knowledge interactions like rooms_per_person beat polynomial features. (2) Log-transforming skewed features. (3) Date/time extraction when temporal patterns exist. (4) GroupBy aggregations when groups are meaningful. (5) Cyclical encoding for periodic features like month and day-of-week.
FEATURE REMOVAL
Removing Useless and Harmful Features
Not all features help — some actively hurt model performance
Adding more features does NOT always help. Irrelevant features add noise to the model's decision boundary. Correlated features cause coefficient instability. ID columns cause overfitting. Knowing what to remove is as important as knowing what to keep.
Features to always remove
· ID columns — unique per row, causes severe overfitting
· Free-text without NLP — too sparse
· URL, email, IP address — too unique
· Timestamp raw — extract year/month/day first
· Constant columns — zero variance
· Duplicate columns
· Target leakage — features derived from the target
Features to consider removing
· High missingness (>50%) without fix
· Near-zero variance
· High cardinality categorical (>50 unique) without encoding
· Highly correlated pairs (|r|>0.90) — keep only one
· Low feature importance in tree models
· Features with VIF > 10
Target leakage — the silent killer
Model trains perfectly but fails in production.

Leakage examples:
· "claim_approved" predicting "fraud" — approval happens AFTER
· "future_salary" predicting "job_change" — uses future data
· "treatment_received" predicting "diagnosis" — backwards causality

Fix: only use features available at prediction time
# ══════════════════════════════════════════════════════════ # FEATURE REMOVAL — All Techniques # ══════════════════════════════════════════════════════════ import numpy as np import pandas as pd from sklearn.feature_selection import VarianceThreshold from sklearn.preprocessing import StandardScaler from sklearn.ensemble import RandomForestClassifier from sklearn.linear_model import LogisticRegression from statsmodels.stats.outliers_influence import variance_inflation_factor import warnings warnings.filterwarnings('ignore') # Create synthetic dataset with various problem features np.random.seed(42) n = 500 base = np.random.randn(n, 5) df = pd.DataFrame({ 'id': range(n), # ID — must remove 'useful_1': base[:,0], 'useful_2': base[:,1], 'useful_3': base[:,2], 'constant': np.ones(n), # constant — must remove 'near_constant':np.where(np.random.rand(n)>0.02, 5, 6),# near constant 'duplicate_1': base[:,0], # duplicate of useful_1 'correlated_95':base[:,0]*0.95 + np.random.randn(n)*0.1,# |r|=0.95 with useful_1 'correlated_80':base[:,1]*0.80 + np.random.randn(n)*0.3,# |r|=0.80 with useful_2 'noisy_random': np.random.randn(n), # pure noise 'noisy_random2':np.random.randn(n), # pure noise 'sparse_50pct': np.where(np.random.rand(n)>0.5, base[:,3], np.nan), 'target': (base[:,0]+base[:,1]+base[:,2] > 0).astype(int) }) print(f"Starting features: {df.shape[1]-1}") # ── Step 1: Remove obvious bad features ─────────────────── print("\n── Step 1: Remove always-bad features ──") # Remove ID-like columns id_cols = ['id'] df.drop(columns=id_cols, inplace=True) print(f" Removed ID columns: {id_cols}") # Remove constant features (zero variance) constant_cols = [c for c in df.columns if df[c].nunique() <= 1] df.drop(columns=constant_cols, inplace=True) print(f" Removed constants: {constant_cols}") # Remove near-constant (VarianceThreshold) X_var = df.drop('target', axis=1).fillna(0) vt = VarianceThreshold(threshold=0.01) vt.fit(X_var) low_var = [c for c, s in zip(X_var.columns, vt.get_support()) if not s] df.drop(columns=low_var, inplace=True) print(f" Removed near-zero variance: {low_var}") # Remove exact duplicates X_check = df.drop('target', axis=1) dup_cols = [] for i, c1 in enumerate(X_check.columns): for c2 in X_check.columns[i+1:]: if X_check[c1].equals(X_check[c2]): dup_cols.append(c2) df.drop(columns=set(dup_cols), inplace=True) print(f" Removed duplicate columns: {dup_cols}") print(f" Features after step 1: {df.shape[1]-1}") # ── Step 2: Correlation-based removal ───────────────────── print("\n── Step 2: Correlation-based removal ──") X = df.drop('target', axis=1).fillna(0) corr_matrix = X.corr() # Find pairs with |r| > 0.85 — keep feature with higher target correlation target_corr = abs(df[X.columns].corrwith(df['target'].astype(float))) to_drop_corr = set() for i in range(len(corr_matrix.columns)): for j in range(i+1, len(corr_matrix.columns)): r = abs(corr_matrix.iloc[i,j]) if r > 0.85: c1, c2 = corr_matrix.columns[i], corr_matrix.columns[j] drop = c1 if target_corr.get(c1, 0) < target_corr.get(c2, 0) else c2 to_drop_corr.add(drop) print(f" High corr: {c1} <-> {c2} r={r:.3f} dropping: {drop}") df.drop(columns=list(to_drop_corr), inplace=True) print(f" Features after correlation filter: {df.shape[1]-1}") # ── Step 3: VIF — multicollinearity ─────────────────────── print("\n── Step 3: VIF — Variance Inflation Factor ──") X_vif = df.drop('target', axis=1).fillna(0) scaler = StandardScaler() X_scaled = pd.DataFrame(scaler.fit_transform(X_vif), columns=X_vif.columns) vif_df = pd.DataFrame({ 'feature': X_scaled.columns, 'VIF': [variance_inflation_factor(X_scaled.values, i) for i in range(X_scaled.shape[1])] }).sort_values('VIF', ascending=False) print(vif_df.to_string(index=False)) print(" VIF > 10 = severe multicollinearity → remove or use Ridge regression") print(" VIF > 5 = moderate multicollinearity → consider removing") high_vif = vif_df[vif_df['VIF'] > 10]['feature'].tolist() if high_vif: print(f" Removing high-VIF features: {high_vif}") df.drop(columns=high_vif, inplace=True) # ── Step 4: Statistical significance ────────────────────── print("\n── Step 4: Statistical feature significance ──") from sklearn.feature_selection import f_classif, mutual_info_classif X_final = df.drop('target', axis=1).fillna(0) y_final = df['target'] f_scores, p_values = f_classif(X_final, y_final) mi_scores = mutual_info_classif(X_final, y_final, random_state=42) significance_df = pd.DataFrame({ 'feature': X_final.columns, 'F_score': f_scores.round(3), 'p_value': p_values.round(4), 'MI': mi_scores.round(4), 'significant': p_values < 0.05 }).sort_values('F_score', ascending=False) print(significance_df.to_string(index=False)) # Remove statistically insignificant features insig = significance_df[~significance_df['significant']]['feature'].tolist() if insig: print(f"\n Statistically insignificant (p>0.05): {insig}") df.drop(columns=insig, inplace=True) # ── Step 5: Tree-based importance ───────────────────────── print("\n── Step 5: Tree-based feature importance ──") X_final = df.drop('target', axis=1).fillna(0) y_final = df['target'] rf = RandomForestClassifier(n_estimators=200, random_state=42, n_jobs=-1) rf.fit(X_final, y_final) importance_df = pd.DataFrame({ 'feature': X_final.columns, 'importance': rf.feature_importances_ }).sort_values('importance', ascending=False) print(importance_df.to_string(index=False)) # Remove near-zero importance features zero_imp = importance_df[importance_df['importance'] < 0.01]['feature'].tolist() print(f"\n Near-zero importance: {zero_imp}") # ── Step 6: Target leakage check ───────────────────────── print("\n── Step 6: Target Leakage Detection ──") print(""" Target leakage = using information that would not be available at the time of prediction in the real world. Leakage red flags: 1. Model accuracy is suspiciously high (>99%) on complex problem 2. Feature has almost perfect correlation with target (|r| > 0.98) 3. Feature is computed AFTER the target event 4. Feature contains the target under a different name 5. Test accuracy much lower than train accuracy How to check: """) # Check for suspiciously high feature-target correlations X_check = df.drop('target', axis=1).fillna(0) feat_target_corr = abs(X_check.corrwith(df['target'].astype(float))) suspicious = feat_target_corr[feat_target_corr > 0.9] if len(suspicious) > 0: print(f" SUSPICIOUS: features with |r|>0.9 with target: {suspicious.to_dict()}") print(" Investigate these — possible leakage!") else: print(f" No obvious leakage detected (max |r| with target = {feat_target_corr.max():.3f})") print(f"\nFinal feature count: {df.shape[1]-1} (started with {n} engineered features)")
Target leakage is the most dangerous bug in ML. The model looks perfect in development but fails completely in production because the leaked feature is not available when making real predictions. Always ask "would I have this feature at prediction time, before the outcome occurs?"
