Situational question cards
Q: I just loaded a new dataset. Where do I start?
Start with shape and dtypes. Then plot histogram/KDE for every numerical feature and sorted bar chart for every categorical feature. This gives you the distribution of each feature in 5 minutes.
→ Histogram, KDE, Bar chart, Missingness heatmap
Q: I need to find which features predict my target.
For numerical features: scatter plot vs target + correlation. For categorical features: box plot grouped by category. Then feature importance from a quick Random Forest.
→ Scatter, Box plot by class, Correlation heatmap, Feature importance
Q: I suspect there are outliers in my data.
Box plots show outliers visually as dots beyond the whiskers. Combine with histogram to see if the distribution is skewed. Then use IQR method or Z-score to quantify them.
→ Box plot, Histogram, Strip plot, Violin plot
Q: I want to know if two features are redundant.
Plot a scatter plot of the two features. If points form a straight line, they are highly correlated and redundant. Confirm with correlation heatmap — look for values above 0.85.
→ Scatter plot, Correlation heatmap, Pair plot
Q: My model is not performing well. What is wrong?
Plot residuals vs fitted — patterns mean the model is missing signal. Learning curve — large train/val gap means overfitting. Confusion matrix — which specific classes are misclassified.
→ Residual plot, Learning curve, Confusion matrix, SHAP
Q: I have a classification problem. Is the data balanced?
Plot a bar chart of target class counts with percentages. If one class has less than 20%, you have class imbalance — use AUC-PR not accuracy, and consider class weights or SMOTE.
→ Target bar chart, Class distribution pie
Q: I need to decide whether to log-transform a feature.
Plot histogram — if it has a long right tail (skewness > 1), apply log transform. Then plot histogram again after transform to confirm it looks more bell-shaped.
→ Histogram before/after, Q-Q plot, Skewness value
Q: I want to see ALL feature relationships at once.
Use a pair plot (scatter matrix) for up to 10 features. Color points by target class to see separability. For more features, use correlation heatmap + top features pair plot.
→ Pair plot, Correlation heatmap, PCA 2D scatter
UNDERSTANDING DISTRIBUTIONS
Visualizing Distributions — Histogram, KDE, Box, Violin, Q-Q
Before any modeling: understand what each feature looks like
Histogram + KDE
Shows shape of distribution. KDE (kernel density estimate) is a smoothed version. Use together.

What to look for:
· Single peak = unimodal (good for linear models)
· Two peaks = bimodal (two sub-populations!)
· Right tail = positively skewed → log transform
· Left tail = negatively skewed
Box plot
Shows: median (line), IQR (box), whiskers (1.5×IQR), outliers (dots beyond whiskers).

Use for: fast outlier detection, comparing distribution across groups.

Limitation: hides bimodal distributions — always combine with histogram.
Violin + Q-Q plot
Violin: box plot + KDE on both sides. Shows full distribution shape including bimodality.

Q-Q plot: tests normality. If points follow the diagonal line = normal. Curves away = skewed. S-shape = heavy tails.
Histogram + KDE — normal vs skewed vs bimodal
Box plots — comparison across groups
Q-Q Plot — testing normality
Violin plot — distribution shape by class

Class distributions that barely overlap → excellent feature for classification.
FINDING RELATIONSHIPS
Scatter Plots, Correlation, Pair Plots, Heatmaps
How features relate to each other and to the target
Scatter plot — two numerical features
Best way to see linear vs non-linear relationships. Add regression line to quantify. Color by target class to see separability. Use hex bin or 2D KDE for large datasets (>10k points) — scatter becomes a black blob.
Correlation heatmap
Shows pairwise linear correlation between ALL numerical features at once. Look for: dark red (r≈1, redundant), dark blue (r≈-1, redundant inverse). Values above 0.85 = feature pair is likely redundant — remove one.
Pair plot (scatter matrix)
Scatter plot of every pair of features in one grid. Diagonal shows distribution. Color by target class. Reveals both relationships AND separability in one view. Limit to 8–10 features — beyond that it becomes too small to read.
Correlation Heatmap — feature redundancy
Scatter — feature vs target (coloured by class)

What to look for in correlation heatmaps: Any pair with |r| > 0.85 is redundant — keep only the one with higher correlation to the target. The target row/column shows which features are most linearly predictive. Note that correlation only captures LINEAR relationships — a feature can have r=0 with the target but still be highly predictive in a non-linear way.

Permutation importance is the most reliable feature importance method. It measures actual AUC drop when a feature is randomly shuffled — if AUC drops a lot, the feature is important. If AUC stays the same or improves, the feature adds noise. Unlike tree importance, it works for any model and is not biased toward high-cardinality features.
