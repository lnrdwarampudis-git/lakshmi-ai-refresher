Error vs Loss — they are NOT the same thing
One of the most commonly confused pairs of terms in ML — understanding the difference is fundamental
Error — the raw difference
Error is the raw difference between a prediction and the true value for a single data point. It is a diagnostic measurement — it tells you how wrong one prediction is.

Formula:
error = y_true − y_predicted

Properties:
· Can be positive or negative
· Measured in the same units as the target
· Exists per data point — not a single number
· Not directly optimized during training

Also called: residual (in linear regression)
Loss — what the model optimizes
Loss is a mathematical function of errors aggregated across the dataset. It is a single number that gradient descent minimizes during training.

Formula:
loss = f(errors across all samples)

Properties:
· Always a single scalar value
· Always non-negative (≥ 0)
· Has smooth gradient — differentiable
· Directly minimized by gradient descent

Examples: MSE, MAE, cross-entropy, Huber
Individual error (residual): eᵢ = yᵢ − ŷᵢ ← one number per sample
MSE loss: L = (1/n) Σ eᵢ² = (1/n) Σ(yᵢ − ŷᵢ)² ← one number for all samples
MAE loss: L = (1/n) Σ|eᵢ| = (1/n) Σ|yᵢ − ŷᵢ| ← one number for all samples
RMSE: √(MSE) ← same units as target
Key distinction: Error is what happened (per prediction). Loss is what the model is trying to fix (the training signal). You report errors to stakeholders (e.g. "our model is off by $200 on average"). You minimize loss during training.

Loss vs Metric — another common confusion
Loss: used during training. Must be differentiable. Gradient descent minimizes it.

Metric: used to evaluate the model after training. Human-interpretable. Does not need to be differentiable.

Example: you train with MSE loss but report RMSE metric to your team (same units as target = more interpretable).
Training loss vs validation loss vs test error
Training loss: loss on training set — what gradient descent minimizes.

Validation loss: loss on held-out data — tells you about overfitting.

Test error: final metric on unseen test set — true generalization performance. Reported to stakeholders.
import numpy as np y_true = np.array([50, 80, 30, 95, 60]) y_pred = np.array([45, 85, 38, 90, 70]) # Individual ERRORS (residuals) — one per sample errors = y_true - y_pred print("Errors:", errors) # [ 5, -5, -8, 5, -10] # LOSS functions — single number, aggregated mse = np.mean(errors**2) # Mean Squared Error mae = np.mean(np.abs(errors)) # Mean Absolute Error rmse = np.sqrt(mse) # Root Mean Squared Error mape = np.mean(np.abs(errors/y_true))*100 # % error print(f"MSE: {mse:.2f}") print(f"MAE: {mae:.2f}") print(f"RMSE: {rmse:.2f}") print(f"MAPE: {mape:.2f}%") # PySpark — Databricks from pyspark.ml.evaluation import RegressionEvaluator for metric in ["rmse", "mae", "r2", "mse"]: val = RegressionEvaluator(metricName=metric, labelCol="label", predictionCol="prediction") print(f"{metric}: {val.evaluate(predictions):.4f}")
