UNSUPERVISED LEARNING — COMPLETE GUIDE
What is Unsupervised Learning?
Finding hidden structure in data without labels — the foundation of data exploration
The key difference from supervised learning: There is no target variable y. The algorithm must discover patterns, groups, or structure purely from the input features X. No right or wrong answer — the quality depends on whether the discovered structure is meaningful and useful.
Supervised vs Unsupervised
Supervised:
· Has labels (X, y)
· Predicts known target
· Clear evaluation metric
· Classification / Regression

Unsupervised:
· Only features (X)
· Discovers hidden structure
· No ground truth metric
· Clustering / Reduction / Anomaly
Three families
Clustering: Group similar data points together. K-Means, DBSCAN, Hierarchical, GMM.

Dimensionality Reduction: Compress features while preserving structure. PCA, t-SNE, UMAP, Autoencoders.

Anomaly Detection: Find rare unusual points. Isolation Forest, One-Class SVM, LOF.
Where unsupervised is used
· Customer segmentation (no known groups)
· Fraud detection (few labelled examples)
· Data exploration before supervised ML
· Recommender systems
· Topic modelling in NLP
· Feature learning for deep models
· Compressing embeddings for RAG
· Finding data quality problems

Unsupervised vs Semi-supervised: If you have a few labels (5–10% of data), use semi-supervised learning (Label Propagation, Self-Training) which combines unsupervised structure with the few available labels. This often beats purely supervised models trained on small labelled data.
CLUSTERING ALGORITHMS
Clustering — Grouping Similar Data Points
K-Means · DBSCAN · Hierarchical · GMM · Choosing k · Evaluation
Choosing the right number of clusters — k selection
Elbow method
Plot inertia (within-cluster sum of squares) vs k. Look for the "elbow" where adding more clusters gives diminishing returns.

inertias = [KMeans(k).fit(X).inertia_ for k in range(1,15)]

Limitation: Elbow is often subtle. Combine with silhouette score.
Silhouette score
Measures how similar each point is to its own cluster vs other clusters. Range: −1 to 1. Higher = better separated clusters.

silhouette_score(X, labels)

Best k = highest silhouette score. Works for any clustering algorithm.
Other metrics
Davies-Bouldin: ratio of within-cluster scatter to between-cluster separation. Lower = better.

Calinski-Harabasz: ratio of between/within cluster dispersion. Higher = better.

Gap statistic: compares inertia to random data. More principled but slower.

Non-spherical shapes (DBSCAN wins) X_moons, y_moons = make_moons(n_samples=400, noise=0.08, random_state=42) # 3. Best DBSCAN config for moons db_best = DBSCAN(eps=0.3, min_samples=5) labels_db = db_best.fit_predict(X_m) n_clust = len(set(labels_db)) - (1 if -1 in labels_db else 0) n_noise = (labels_db == -1).sum() print(f"\nBest DBSCAN: {n_clust} clusters, {n_noise} noise points") print(f"ARI vs true: {adjusted_rand_score(y_moons, labels_db):.4f}") # show best comparison print(f" linkage={linkage:9s} k={k}: silhouette={sil:.4f} ARI={ari:.4f}") print("\nWhen to use hierarchical:") print(" ✓ Don't know k in advance (read dendrogram)") print(" ✓ Want to see the full hierarchy") print(" ✓ Small dataset (<5000 rows)") print(" ✗ Large datasets (O(n²) memory)") # ══════════════════════════════════════════════ # GAUSSIAN MIXTURE MODEL # ══════════════════════════════════════════════ print("\n" + "="*55) print("GAUSSIAN MIXTURE MODEL (GMM)") print("="*55) # Model selection using BIC/AIC print("BIC/AIC for model selection:") for k in range(1, 8): gmm = GaussianMixture(n_components=k, random_state=42, n_init=5) gmm.fit(X_h) bic = gmm.bic(X_h) aic = gmm.aic(X_h) print(f" k={k}: BIC={bic:.1f} AIC={aic:.1f}") best_gmm = GaussianMixture(n_components=3, random_state=42, n_init=10) labels_gmm = best_gmm.fit_predict(X_h) probs = best_gmm.predict_proba(X_h) print(f"\nGMM k=3:") print(f" ARI vs true: {adjusted_rand_score(iris.target, labels_gmm):.4f}") print(f" Avg max probability: {probs.max(axis=1).mean():.4f}") print(f" Points with p<0.7 (uncertain): {(probs.max(axis=1)<0.7).sum()}") print("\nGMM advantage: SOFT assignments — probability of belonging to each cluster") print("K-Means: hard assignments only (binary)") # Clustering evaluation is hard because there is no ground truth. Use multiple metrics together — silhouette score for separation quality, Davies-Bouldin for cluster compactness, and always visualise with PCA or t-SNE reduced to 2D. If the clusters look meaningful to a domain expert, they are good clusters.
DIMENSIONALITY REDUCTION
Dimensionality Reduction — Compressing Features Intelligently
PCA · t-SNE · UMAP · Autoencoders · Feature Compression for ML
Why reduce dimensions?
· Curse of dimensionality — distance metrics break in high-D
· Visualisation — only works in 2D/3D
· Noise removal — discard low-variance components
· Faster training — fewer features = faster ML
· Better clustering — cluster in reduced space
· Compression — store/transmit less data
Linear vs non-linear
Linear (PCA, SVD): finds straight axes. Fast, deterministic, invertible. Best for preprocessing before ML.

Non-linear (t-SNE, UMAP): preserves local neighbourhoods. Reveals clusters that PCA misses. Better for visualisation. Cannot be applied to new data easily (t-SNE).
When to use each
PCA: preprocessing before ML, noise removal, explained variance
t-SNE: visualisation ONLY — clusters in 2D
UMAP: visualisation + preprocessing (faster than t-SNE, preserves global)
Autoencoder: non-linear compression, anomaly detection

Choose n_components for 95% variance n_95 = np.argmax(cum_var >= 0.95) + 1 n_99 = np.argmax(cum_var >= 0.99) + 1 print(f"\n95% variance needs: {n_95} components (from 64)") print(f"99% variance needs: {n_99} components (from 64)") # keep 95% variance ('clf', LogisticRegression(max_iter=1000)) ]) auc_pipe = cross_val_score(pca_pipe, cancer.data, y_c, cv=5, scoring='roc_auc').mean() print(f"\nPCA Pipeline (95% var): AUC={auc_pipe:.4f}") pca_pipe.fit(cancer.data, y_c) print(f" n_components selected: {pca_pipe['pca'].n_components_}") # NMF (parts-based, interpretable) X_pos = np.abs(StandardScaler().fit_transform(digits.data)) nmf = NMF(n_components=20, random_state=42, max_iter=500) X_nmf = nmf.fit_transform(X_pos) print(f"\nNMF: {X_pos.shape} → {X_nmf.shape}") print(" USE WHEN: non-negative data (images, counts, TF-IDF)") print(" Components are additive parts — more interpretable than PCA") # The single most important rule about t-SNE: Never use t-SNE features as input to a machine learning model. It is a visualisation tool only. Use PCA instead for preprocessing. UMAP is the only non-linear method you can use for both visualisation and ML preprocessing.
ANOMALY DETECTION
Anomaly Detection — Finding the Unusual
Isolation Forest · One-Class SVM · LOF · Autoencoder · Practical guide
What is anomaly detection?
Finding data points that are significantly different from the majority. Unlike fraud classification, anomaly detection works WITHOUT labelled fraud examples.

Use cases:
· Fraud detection with no labels
· Network intrusion detection
· Manufacturing defect detection
· Data quality monitoring
· Medical rare condition screening
Three approaches
Statistical: points far from distribution. Z-score, IQR, Gaussian envelope.

Isolation-based: points that are easy to isolate. Isolation Forest — fast and accurate.

Density-based: points in sparse regions. LOF, HDBSCAN noise points.

Reconstruction-based: points the model cannot reconstruct. Autoencoders.
Evaluation (no labels)
Hard to evaluate without labels. Practical approaches:

· Plot anomaly scores — look for clear separation
· Inject known synthetic anomalies and check detection rate
· Have domain expert review flagged points
· If some labels exist: precision@k (top k flagged = how many real?)

contamination parameter = expected % anomalies
# ══════════════════════════════════════════════════════════ # ANOMALY DETECTION — All Methods # ══════════════════════════════════════════════════════════ import numpy as np import pandas as pd from sklearn.ensemble import IsolationForest from sklearn.neighbors import LocalOutlierFactor from sklearn.svm import OneClassSVM from sklearn.covariance import EllipticEnvelope from sklearn.preprocessing import StandardScaler from sklearn.datasets import make_blobs from sklearn.metrics import roc_auc_score, average_precision_score import warnings; warnings.filterwarnings('ignore') np.random.seed(42) # Create dataset with known anomalies X_normal = np.random.normal(0, 1, (400, 2)) X_anom = np.random.uniform(-5, 5, (20, 2)) # 5% anomalies X_all = np.vstack([X_normal, X_anom]) y_true = np.array([1]*400 + [-1]*20) # 1=normal, -1=anomaly X_sc = StandardScaler().fit_transform(X_all) contamination = 0.05 print("="*55) print("ANOMALY DETECTION COMPARISON") print("="*55) print(f"Dataset: {len(X_normal)} normal + {len(X_anom)} anomalies") # All detectors detectors = { 'Isolation Forest': IsolationForest(contamination=contamination, random_state=42, n_jobs=-1), 'Local Outlier Factor':LocalOutlierFactor(n_neighbors=20, contamination=contamination), 'One-Class SVM': OneClassSVM(kernel='rbf', nu=contamination, gamma='scale'), 'Elliptic Envelope': EllipticEnvelope(contamination=contamination, random_state=42), } print(f"\n{'Method':25s} {'Precision':10s} {'Recall':8s} {'F1':7s} {'AUC':7s}") print("-"*65) results_anom = {} for name, detector in detectors.items(): if isinstance(detector, LocalOutlierFactor): y_pred = detector.fit_predict(X_sc) scores = -detector.negative_outlier_factor_ else: detector.fit(X_sc) y_pred = detector.predict(X_sc) scores = detector.score_samples(X_sc) if hasattr(detector,'score_samples') else None # Convert: -1=anomaly → 1 for anomaly evaluation tp = np.sum((y_pred==-1) & (y_true==-1)) fp = np.sum((y_pred==-1) & (y_true==1)) fn = np.sum((y_pred==1) & (y_true==-1)) prec = tp/(tp+fp+1e-9); rec = tp/(tp+fn+1e-9) f1 = 2*prec*rec/(prec+rec+1e-9) auc = roc_auc_score((y_true==-1).astype(int), -scores) if scores is not None else 0 results_anom[name] = {'prec':prec,'rec':rec,'f1':f1,'auc':auc} print(f"{name:25s} {prec:10.4f} {rec:8.4f} {f1:7.4f} {auc:7.4f}") # ── Isolation Forest deep dive ───────────────────────────── print("\n" + "="*55) print("ISOLATION FOREST — Deep Dive") print("="*55) print(""" How it works: 1. Randomly select a feature 2. Randomly select a split value between min and max 3. Anomalies need FEWER splits to isolate (shorter path length) 4. Normal points need MORE splits (longer path) 5. Anomaly score = average path length over many trees contamination parameter = expected % of anomalies in data Start with 0.05 (5%), adjust based on domain knowledge """) iso = IsolationForest(contamination=0.05, n_estimators=200, random_state=42) iso.fit(X_sc) scores = iso.score_samples(X_sc) print("Score interpretation:") print(f" Most normal score: {scores[y_true==1].max():.4f}") print(f" Avg normal score: {scores[y_true==1].mean():.4f}") print(f" Avg anomaly score: {scores[y_true==-1].mean():.4f}") print(f" Most anomalous: {scores.min():.4f}") print(f" Scores near 0 = ambiguous; strongly negative = definite anomaly") # contamination sensitivity print("\ncontamination parameter effect:") for c in [0.01, 0.02, 0.05, 0.10, 0.15, 0.20]: iso_c = IsolationForest(contamination=c, random_state=42, n_jobs=-1) pred = iso_c.fit_predict(X_sc) n_flagged = (pred==-1).sum() tp = np.sum((pred==-1) & (y_true==-1)) fp = np.sum((pred==-1) & (y_true==1)) print(f" contamination={c:.2f}: flagged={n_flagged:4d} TP={tp} FP={fp}") # ── LOF deep dive ───────────────────────────────────────── print("\n" + "="*55) print("LOCAL OUTLIER FACTOR (LOF)") print("="*55) print(""" How it works: Compares local density of a point to its k-nearest neighbours. LOF > 1 = lower density than neighbours = anomaly LOF ≈ 1 = similar density = normal n_neighbors: smaller = more local; larger = more global USE WHEN: anomalies are local (normal in one region, anomalous in another) """) for k in [5, 10, 15, 20, 30, 50]: lof = LocalOutlierFactor(n_neighbors=k, contamination=0.05) pred = lof.fit_predict(X_sc) tp = np.sum((pred==-1) & (y_true==-1)) fp = np.sum((pred==-1) & (y_true==1)) f1_v = 2*tp/(2*tp+fp+(20-tp)+1e-9) print(f" n_neighbors={k:3d}: TP={tp:2d} FP={fp:3d} F1={f1_v:.4f}") # ── Real-world pipeline ──────────────────────────────────── print("\n" + "="*55) print("REAL-WORLD ANOMALY DETECTION PIPELINE") print("="*55) print(""" Step 1: EDA — understand what normal looks like Step 2: Feature engineering — create relevant features Step 3: Scale features (StandardScaler or RobustScaler) Step 4: Train Isolation Forest with contamination = expected % Step 5: Get anomaly scores — rank all points Step 6: Set threshold — business decision (top 1%? 5%?) Step 7: Review top flagged points with domain expert Step 8: Tune contamination parameter based on feedback Step 9: If labels become available — switch to supervised fraud detection """) # Delta table pipeline (Databricks) print("Databricks / PySpark pipeline:") print(""" from sklearn.ensemble import IsolationForest import pandas as pd # Load from Delta table df = spark.table("catalog.schema.transactions").toPandas() # Feature selection and scaling features = ['amount', 'velocity', 'hour', 'distance'] X = StandardScaler().fit_transform(df[features].fillna(0)) # Train isolation forest iso = IsolationForest(contamination=0.01, # expect 1% fraud n_estimators=200, random_state=42, n_jobs=-1) iso.fit(X) # Score all transactions df['anomaly_score'] = iso.score_samples(X) # lower = more anomalous df['is_anomaly'] = iso.predict(X) == -1 # True/False flag # Return top 1% most anomalous for review top_anomalies = df.nsmallest(int(len(df)*0.01), 'anomaly_score') print(f"Flagged for review: {len(top_anomalies)} transactions") # Write back to Delta spark.createDataFrame(df).write.format("delta").mode("overwrite") .saveAsTable("catalog.schema.transactions_scored") """)
The contamination parameter is the most important decision in anomaly detection. Too low = you miss real anomalies. Too high = you waste analyst time on false positives. Start with your domain knowledge (if 1% of transactions are fraud, set contamination=0.01). Adjust based on analyst feedback.
RIGHT SITUATION → RIGHT ALGORITHM
Which Unsupervised Algorithm for Which Situation
Real-world scenarios mapped to the best algorithm choice
Clustering situations
I want to segment customers into groups — I don't know how many groups exist
Use K-Means. Try k from 2 to 10, plot silhouette score vs k, pick the k where silhouette is highest. Then profile each cluster — what makes them different? Those differences are your customer segments.
→ K-Means + elbow + silhouette score + cluster profiling
My clusters are irregular shapes — not round blobs
K-Means assumes spherical clusters. Use DBSCAN instead — it finds arbitrarily shaped clusters. Set eps (neighbourhood radius) and min_samples. Points that don't fit any cluster are labelled as noise (-1).
→ DBSCAN → HDBSCAN (more robust, auto-eps)
I don't know k at all — I want to discover the natural hierarchy
Hierarchical/Agglomerative clustering builds a full tree (dendrogram) from individual points up to one big cluster. You choose k by cutting the dendrogram at any level — no need to decide k before training.
→ AgglomerativeClustering + dendrogram + ward linkage
I need soft cluster assignments — probability of belonging to each cluster
K-Means gives hard assignments only. Use Gaussian Mixture Model (GMM) — every point gets a probability for each cluster. Useful when points naturally overlap between clusters (e.g. customers in transition between segments).
→ GaussianMixture + predict_proba() + BIC for k selection
My data has very different densities in different regions
K-Means and DBSCAN with a single eps parameter fail on variable-density data. Use HDBSCAN which adapts to local density automatically. It is DBSCAN with a minimum spanning tree-based approach.
→ HDBSCAN (pip install hdbscan)
I have millions of rows — clustering is too slow
K-Means scales linearly with data. For very large datasets use MiniBatchKMeans which processes random mini-batches instead of full data. 10-100x faster with small accuracy trade-off. Works in Spark MLlib too.
→ MiniBatchKMeans → Spark MLlib KMeans
I want to cluster documents or text
Convert text to TF-IDF or embeddings first. Then cluster. K-Means on TF-IDF vectors works well for topic discovery. Use cosine distance not euclidean for TF-IDF. LDA gives interpretable topic-word distributions.
→ TF-IDF + K-Means (cosine) → LDA topic model → BERT embeddings + K-Means
I want to improve my supervised model using cluster labels as features
Train K-Means first, then add the cluster label as a new feature to your supervised model. This gives the model information about which "region" of the feature space each sample belongs to — often improves accuracy.
→ K-Means → add cluster_id as feature → XGBoost
Dimensionality reduction situations
I have 100+ features and want to reduce before ML training
Use PCA. Set n_components to keep 95% of variance. This removes noise, reduces training time, and can improve accuracy by eliminating irrelevant components. Always scale features first.
→ StandardScaler → PCA(n_components=0.95) → ML model
I want to visualise my high-dimensional data in 2D
Use t-SNE for the best cluster visualisation. First reduce to 50 dimensions with PCA (faster), then apply t-SNE to get 2D. Color points by known labels to check if clusters align with reality.
→ PCA(50 dims) → t-SNE(2D) → scatter plot colored by class
I have sparse text/TF-IDF data (many zero values)
PCA does not work on sparse matrices efficiently. Use TruncatedSVD (also called LSA — Latent Semantic Analysis) which works directly on sparse matrices without centering the data first.
→ TruncatedSVD(n_components=100) → downstream ML
I suspect my model is missing non-linear structure in high dimensions
PCA only finds linear structure. Use UMAP for non-linear reduction that preserves both local and global structure. Unlike t-SNE, UMAP can transform new data and is much faster.
→ UMAP(n_components=20) → ML model (treats as preprocessing)
I want to understand which original features explain the most variance
PCA components are linear combinations of original features. Plot PCA loadings — high absolute loading on a component means that feature strongly drives that component. Use explained_variance_ratio_ to rank components.
→ PCA → plot loadings heatmap → interpret components
Anomaly detection situations
I need to detect fraud but I have NO labelled fraud examples
This is the classic unsupervised anomaly detection problem. Use Isolation Forest with contamination = expected fraud rate. Get anomaly scores for all transactions, rank them, review the top k% with analysts.
→ IsolationForest(contamination=0.01) → rank by score → review top 1%
I have some labelled anomalies (10-50 examples) — mostly normal data
Semi-supervised: train One-Class SVM on normal data only (exclude anomalies). Or use the few labels to set a better threshold for Isolation Forest scores. If enough labels, switch to supervised fraud detection.
→ OneClassSVM on normal data → threshold using labelled examples
I want to monitor data quality in my Delta table pipeline
Train Isolation Forest on historical clean data. Score each new batch as it arrives. Flag rows with scores below threshold for investigation. This catches data drift, schema changes, and upstream data quality issues automatically.
→ IsolationForest on historical → score new batches → alert on low scores
My anomalies are local — normal globally but anomalous in their neighbourhood
Isolation Forest is a global method. Use Local Outlier Factor (LOF) which compares local density. A point with much lower density than its neighbours is flagged — even if it looks normal globally.
→ LocalOutlierFactor(n_neighbors=20) → score_samples()

# ══════════════════════════════════════════════════════════════ # COMPLETE UNSUPERVISED LEARNING WORKFLOW # K-Means + DBSCAN + PCA + t-SNE + Isolation Forest + LDA # Datasets: Iris, Digits, 20Newsgroups, California Housing # ══════════════════════════════════════════════════════════════ import numpy as np import pandas as pd from sklearn.datasets import (load_iris, load_digits, fetch_california_housing, fetch_20newsgroups) from sklearn.preprocessing import StandardScaler from sklearn.pipeline import Pipeline import warnings; warnings.filterwarnings('ignore') # ══════════════════════════════════════════════ # PHASE 1: CLUSTERING — Customer Segmentation # ══════════════════════════════════════════════ print("▶ PHASE 1: CLUSTERING") from sklearn.cluster import (KMeans, DBSCAN, AgglomerativeClustering, MiniBatchKMeans) from sklearn.mixture import GaussianMixture from sklearn.metrics import silhouette_score, davies_bouldin_score from sklearn.metrics import adjusted_rand_score # Using Iris as customer data analogy iris = load_iris(as_frame=True) X = StandardScaler().fit_transform(iris.data) y_true = iris.target.values # Step 1: Find optimal k for K-Means print("\nStep 1: K selection") best_sil, best_k = 0, 2 for k in range(2, 8): labels = KMeans(n_clusters=k, random_state=42, n_init=10).fit_predict(X) sil = silhouette_score(X, labels) db = davies_bouldin_score(X, labels) ari = adjusted_rand_score(y_true, labels) marker = '← BEST' if sil > best_sil else '' print(f" k={k}: silhouette={sil:.4f} DB={db:.4f} ARI={ari:.4f} {marker}") if sil > best_sil: best_sil, best_k = sil, k print(f" Best k = {best_k}") # Step 2: Train best K-Means km = KMeans(n_clusters=best_k, random_state=42, n_init=20) labels_km = km.fit_predict(X) print(f"\nK-Means (k={best_k}):") print(f" Cluster sizes: {dict(zip(*np.unique(labels_km,return_counts=True)))}") print(f" ARI vs true: {adjusted_rand_score(y_true, labels_km):.4f}") # Step 3: Profile clusters df_cluster = iris.data.copy() df_cluster['cluster'] = labels_km print("\nCluster profiles (mean values):") print(df_cluster.groupby('cluster').mean().round(3)) # Step 4: Compare all clustering algorithms print("\nClustering algorithm comparison:") algorithms = { 'K-Means': KMeans(n_clusters=3, random_state=42, n_init=10), 'MiniBatchKMeans':MiniBatchKMeans(n_clusters=3, random_state=42, n_init=10), 'Agglomerative': AgglomerativeClustering(n_clusters=3, linkage='ward'), 'GMM': GaussianMixture(n_components=3, random_state=42), 'DBSCAN': DBSCAN(eps=0.5, min_samples=5), } for name, alg in algorithms.items(): labels = alg.fit_predict(X) n_clust = len(set(labels)) - (1 if -1 in labels else 0) if n_clust > 1 and len(set(labels[labels!=-1])) > 1: noise = (labels==-1).sum() valid = labels != -1 sil = silhouette_score(X[valid], labels[valid]) if valid.sum()>1 else 0 ari = adjusted_rand_score(y_true, labels) print(f" {name:18s}: k={n_clust} sil={sil:.4f} ARI={ari:.4f} noise={noise}") # ══════════════════════════════════════════════ # PHASE 2: DIMENSIONALITY REDUCTION # ══════════════════════════════════════════════ print("\n▶ PHASE 2: DIMENSIONALITY REDUCTION") from sklearn.decomposition import PCA, TruncatedSVD, NMF from sklearn.manifold import TSNE from sklearn.linear_model import LogisticRegression from sklearn.model_selection import cross_val_score digits = load_digits() X_d = StandardScaler().fit_transform(digits.data) # 1797 × 64 y_d = digits.target # Step 1: PCA explained variance pca_full = PCA(random_state=42).fit(X_d) cum_var = np.cumsum(pca_full.explained_variance_ratio_) print("\nPCA variance thresholds:") for threshold in [0.80, 0.90, 0.95, 0.99]: n = np.argmax(cum_var >= threshold) + 1 print(f" {threshold*100:.0f}% variance → {n:3d}/{X_d.shape[1]} components") # Step 2: PCA impact on ML print("\nPCA impact on Logistic Regression (digits 10-class):") lr = LogisticRegression(max_iter=500, random_state=42) base = cross_val_score(lr, X_d, y_d, cv=5).mean() print(f" All {X_d.shape[1]} features: acc={base:.4f}") for n_comp in [5, 10, 20, 30, 40, 50]: X_pca = PCA(n_components=n_comp, random_state=42).fit_transform(X_d) acc = cross_val_score(lr, X_pca, y_d, cv=5).mean() delta = acc - base print(f" {n_comp:3d} components: acc={acc:.4f} delta={delta:+.4f}") # Step 3: t-SNE visualisation quality print("\nt-SNE cluster quality check:") X_pca50 = PCA(n_components=50, random_state=42).fit_transform(X_d) tsne = TSNE(n_components=2, perplexity=30, random_state=42, n_iter=1000) X_tsne = tsne.fit_transform(X_pca50) # Check how well classes separate in 2D from sklearn.cluster import KMeans as KM labels_tsne = KM(n_clusters=10,random_state=42,n_init=10).fit_predict(X_tsne) ari_tsne = adjusted_rand_score(y_d, labels_tsne) print(f" t-SNE 2D → K-Means ARI vs true labels: {ari_tsne:.4f}") print(f" (ARI close to 1 = clusters are well separated in 2D)") # Step 4: PCA for preprocessing pipeline print("\nPCA preprocessing pipeline:") pca_pipe = Pipeline([ ('pca', PCA(n_components=0.95, random_state=42)), ('clf', LogisticRegression(max_iter=500, random_state=42)) ]) acc_pipe = cross_val_score(pca_pipe, X_d, y_d, cv=5).mean() pca_pipe.fit(X_d, y_d) n_selected = pca_pipe['pca'].n_components_ print(f" PCA (95% var) → {n_selected} components → LR accuracy: {acc_pipe:.4f}") # ══════════════════════════════════════════════ # PHASE 3: ANOMALY DETECTION # ══════════════════════════════════════════════ print("\n▶ PHASE 3: ANOMALY DETECTION") from sklearn.ensemble import IsolationForest from sklearn.neighbors import LocalOutlierFactor from sklearn.covariance import EllipticEnvelope # Simulate transaction data with anomalies np.random.seed(42) n_normal, n_anom = 1000, 50 X_normal = np.random.multivariate_normal( mean=[0,0,0,0], cov=[[1,0.5,0.2,0.1],[0.5,1,0.3,0.2],[0.2,0.3,1,0.4],[0.1,0.2,0.4,1]], size=n_normal) X_anom = np.random.uniform(-4, 4, (n_anom, 4)) X_all = np.vstack([X_normal, X_anom]) y_true_anom = np.array([1]*n_normal + [-1]*n_anom) X_all_sc = StandardScaler().fit_transform(X_all) print(f"\nDataset: {n_normal} normal + {n_anom} anomalies ({n_anom/(n_normal+n_anom)*100:.1f}%)") contamination = n_anom / (n_normal + n_anom) detectors = { 'Isolation Forest': IsolationForest(contamination=contamination,n_estimators=200,random_state=42), 'LOF': LocalOutlierFactor(n_neighbors=20,contamination=contamination), 'Elliptic Envelope': EllipticEnvelope(contamination=contamination,random_state=42), } from sklearn.metrics import f1_score, precision_score, recall_score print(f"\n{'Method':22s} {'Precision':10s} {'Recall':8s} {'F1':7s}") for name, det in detectors.items(): if isinstance(det, LocalOutlierFactor): pred = det.fit_predict(X_all_sc) else: pred = det.fit(X_all_sc).predict(X_all_sc) # Convert: sklearn uses -1=anomaly; we need 1=anomaly for metrics y_pred_binary = (pred == -1).astype(int) y_true_binary = (y_true_anom == -1).astype(int) p = precision_score(y_true_binary, y_pred_binary, zero_division=0) r = recall_score(y_true_binary, y_pred_binary, zero_division=0) f = f1_score(y_true_binary, y_pred_binary, zero_division=0) print(f" {name:22s} {p:10.4f} {r:8.4f} {f:7.4f}") # Isolation Forest score analysis iso = IsolationForest(contamination=contamination, n_estimators=300, random_state=42) iso.fit(X_all_sc) scores = iso.score_samples(X_all_sc) print(f"\nAnomaly score analysis:") print(f" Normal mean score: {scores[:n_normal].mean():.4f}") print(f" Anomaly mean score: {scores[n_normal:].mean():.4f}") print(f" Score gap: {scores[:n_normal].mean()-scores[n_normal:].mean():.4f}") print(f" Threshold at 95th pct normal: {np.percentile(scores[:n_normal],5):.4f}") # Top 10 most anomalous top_idx = scores.argsort()[:10] print(f"\nTop 10 most anomalous points:") for rank, idx in enumerate(top_idx): true_label = 'ANOMALY' if y_true_anom[idx]==-1 else 'normal' print(f" #{rank+1:2d} idx={idx:4d} score={scores[idx]:.4f} true={true_label}") # ══════════════════════════════════════════════ # PHASE 4: TOPIC MODELLING (NLP) # ══════════════════════════════════════════════ print("\n▶ PHASE 4: TOPIC MODELLING") from sklearn.decomposition import LatentDirichletAllocation from sklearn.feature_extraction.text import TfidfVectorizer, CountVectorizer cats = ['alt.atheism','comp.graphics','sci.med','soc.religion.christian'] news = fetch_20newsgroups(subset='train', categories=cats, remove=('headers','footers','quotes')) print(f"\n20 Newsgroups: {len(news.data)} documents, {len(cats)} true categories") # LDA topic modelling cv = CountVectorizer(max_features=2000, stop_words='english', min_df=5) X_text = cv.fit_transform(news.data) lda = LatentDirichletAllocation(n_components=4, random_state=42, max_iter=20, learning_method='batch') lda.fit(X_text) # Show top words per topic feature_names = cv.get_feature_names_out() print("\nTop 8 words per LDA topic:") for i, topic in enumerate(lda.components_): top_words = [feature_names[j] for j in topic.argsort()[-8:][::-1]] print(f" Topic {i}: {' | '.join(top_words)}") # Evaluate topic quality doc_topics = lda.transform(X_text) assigned_topics = doc_topics.argmax(axis=1) ari_lda = adjusted_rand_score(news.target, assigned_topics) print(f"\nLDA topic-to-class ARI: {ari_lda:.4f}") print(" (ARI near 1 = topics match true categories well)") # Topic coherence (simple word co-occurrence check) print("\nDocument-topic distribution sample (first 5 docs):") for i in range(5): top_topic = doc_topics[i].argmax() print(f" Doc {i}: most likely Topic {top_topic} ({doc_topics[i][top_topic]:.3f})") print("\n✅ Complete unsupervised learning workflow done!") print("\n── KEY TAKEAWAYS ──") print(" Clustering: K-Means + silhouette for k selection + cluster profiling") print(" Dim reduce: PCA (95% variance) before ML + t-SNE for visualisation only") print(" Anomaly: Isolation Forest with contamination = expected anomaly rate") print(" NLP: LDA topics → compare with domain knowledge")
