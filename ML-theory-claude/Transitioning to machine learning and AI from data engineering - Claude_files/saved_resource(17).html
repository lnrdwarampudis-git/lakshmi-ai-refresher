Model Parameters vs Hyperparameters
The most important distinction in ML — understanding this changes how you approach every model
Model Parameters — learned automatically
These are the internal values the model learns from data during training via gradient descent. You never set these manually — the optimizer finds the best values.

Examples:
· Weights W in every neural network layer
· Biases b in every layer
· Coefficients in linear/logistic regression
· Split thresholds in decision trees
· Centroids in k-means clustering
· Embedding vectors in word2vec/LLMs

Stored in: model checkpoint file (.pt, .pkl, .parquet)
Count: GPT-4 ≈ 1.8 trillion · BERT ≈ 110 million · Your logistic regression = n_features + 1
Hyperparameters — you set before training
These are the configuration choices you make before training starts. They control how learning happens, the model architecture, and training dynamics — but the model does not learn them.

Examples:
· Learning rate (most important!)
· Number of layers and neurons per layer
· Batch size
· Number of epochs
· Regularization strength λ
· Dropout rate
· Number of trees in Random Forest
· Max depth of trees
· k in k-nearest neighbours

Tuned on: validation set only — never test set
Logged in: MLflow params
The one-line rule: If gradient descent updates it → it's a parameter. If you type it in your code before training → it's a hyperparameter.

Two types of hyperparameters
Model architecture HPs: Define the structure — number of layers, neurons, depth. Change these and you get a completely different model.

Training HPs: Control the learning process — learning rate, batch size, epochs, optimizer. Same architecture, different training dynamics.
Why hyperparameters matter so much
A great model with wrong hyperparameters will fail completely. Learning rate too high → loss diverges. Too low → never converges in reasonable time. Batch size too large → poor generalization. The right hyperparameters can be the difference between 70% and 95% accuracy on the same model.
# Parameters vs hyperparameters — side by side in PyTorch import torch.nn as nn # HYPERPARAMETERS — you set these lr = 0.001 # how fast to learn batch_size = 32 # samples per update n_epochs = 50 # how many full passes n_layers = 3 # model architecture hidden_dim = 128 # neurons per layer dropout = 0.2 # regularization # MODEL — architecture defined by hyperparameters model = nn.Sequential( nn.Linear(10, hidden_dim), # 10×128 + 128 = 1408 PARAMETERS (learned) nn.ReLU(), nn.Dropout(dropout), nn.Linear(hidden_dim, 1) # 128×1 + 1 = 129 PARAMETERS (learned) ) # Total parameters: 1408 + 129 = 1537 (all learned by gradient descent) print(sum(p.numel() for p in model.parameters())) # → 1537 # Log hyperparameters — NEVER parameters import mlflow mlflow.log_param("learning_rate", lr) mlflow.log_param("batch_size", batch_size) mlflow.log_param("hidden_dim", hidden_dim) # model.state_dict() = all 1537 learned weights (logged as artifact)
