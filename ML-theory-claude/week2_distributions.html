The normal distribution is everywhere in ML. Why? The Central Limit Theorem says sample means always become normal — so model errors, noise, and residuals are all normally distributed. MSE loss assumes your errors are normal. Z-score normalization rescales your features to μ=0, σ=1 — the standard normal. In PySpark: StandardScaler(withMean=True, withStd=True)

Bernoulli models yes/no outcomes — churn or not, click or not, fraud or not. Logistic regression outputs a Bernoulli probability. The entropy peaks at p=0.5 (maximum uncertainty) and goes to 0 at p=0 or p=1 (certainty). Binary cross-entropy loss is derived directly from Bernoulli log-likelihood.

Binomial = n independent Bernoulli trials. Increase n to 30 — watch it become bell-shaped (Central Limit Theorem in action!). In ML: models that output class probabilities use this thinking. In PySpark: sampling a Delta table k times from n rows follows a binomial distribution.
Which distribution → which ML loss function:

Normal errors → MSE loss (linear/ridge regression)
Bernoulli outcomes → Binary cross-entropy (logistic regression, binary classifiers)
Categorical outcomes → Categorical cross-entropy (softmax classifiers)
Heavy-tailed errors → Huber loss (robust regression)

In Databricks: check your target column distribution before choosing a loss. df.select(F.skewness("target"), F.kurtosis("target")).show() — high kurtosis = heavy tails = use Huber or log-transform first.
# PySpark: identify your target distribution
from pyspark.sql import functions as F
df.select(
  F.mean("target"),
  F.stddev("target"),
  F.skewness("target"), # >1 = right skewed
  F.kurtosis("target") # >3 = heavy tails
).show()
