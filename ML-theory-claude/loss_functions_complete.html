Loss Functions — complete guide
What they are, why they exist, and the full taxonomy
What is a loss function? A loss function measures how wrong a model's prediction is. It converts the difference between predicted and actual values into a single number that gradient descent then minimises. Choosing the right loss function is as important as choosing the right model.
Loss vs Error vs Metric — clarified
Error = raw difference for one sample: eᵢ = yᵢ − ŷᵢ

Loss = aggregated function of errors used during training. Must be differentiable (gradient descent needs ∂L/∂w).

Metric = how you evaluate the model after training. Human-interpretable. Does NOT need to be differentiable.

Example: train with cross-entropy loss, report accuracy metric to stakeholders.
Why different loss functions exist
Different problems have different statistical assumptions about errors:

· Errors normally distributed → MSE
· Binary yes/no outcome → Binary cross-entropy
· Multiple categories → Categorical cross-entropy
· Outliers present → MAE or Huber
· Comparing distributions → KL divergence
· Ranking/similarity → Contrastive / Triplet

The loss function IS the statistical model of your problem.

Binary Cross-Entropy — deep dive
L = −[y·log(p) + (1−y)·log(1−p)]
y=1: L = −log(p) → loss → 0 as p → 1
y=0: L = −log(1−p) → loss → 0 as p → 0

Derived from Bernoulli MLE. Pairs with sigmoid output.

Logistic regression Binary classification Churn, fraud detection
Categorical Cross-Entropy — deep dive
L = −Σ yᵢ · log(p̂ᵢ) over all classes
Only the true class contributes (yᵢ=1 for true class, 0 for rest).
So: L = −log(p̂_true_class)

Pairs with softmax output layer.

Multi-class classification Image classification NLP tasks
Focal Loss — for class imbalance
L = −(1−p)^γ · log(p)
Reduces loss for easy examples (high p) so the model focuses on hard, misclassified examples.

γ=0 → standard BCE. γ=2 → strong focus on hard examples.

Class imbalance Object detection (YOLO) Fraud (1% positive rate)
Hinge Loss — for SVMs
L = max(0, 1 − y · ŷ) where y ∈ {-1, +1}
Penalises predictions within the margin. Zero loss when correct and confident (|ŷ| > 1).

Creates maximum-margin classifier (SVM).

SVM classifier Ranking tasks Not for probabilities
KL Divergence + Specialised Loss Functions
For comparing distributions, generative models, embeddings and ranking
KL Divergence — deep dive
KL(P‖Q) = Σ P(x) · log(P(x)/Q(x))
Measures how much distribution Q differs from distribution P.

Asymmetric: KL(P‖Q) ≠ KL(Q‖P)
Always ≥ 0. Zero only when P = Q exactly.

Used in: VAEs (encoder output vs standard normal), knowledge distillation (student vs teacher), RL (policy updates), attention regularisation.

VAE training Model distillation RL policy gradient
Triplet Loss — deep dive
L = max(0, d(a,p) − d(a,n) + margin)
a = anchor, p = positive (same class), n = negative (different class)

Forces: distance(anchor, positive) + margin < distance(anchor, negative)

Creates an embedding space where similar items cluster together and dissimilar items are pushed apart.

Face recognition Sentence embeddings Product recommendations
ELBO — Variational Autoencoder
ELBO = E[log p(x|z)] − KL(q(z|x)‖p(z))
Two terms:
1. Reconstruction loss: how well decoder recreates input
2. KL term: how close encoder distribution is to standard normal

The KL term acts as regularisation — prevents the encoder from memorising exact points.

Image generation Anomaly detection Latent space learning
Dice Loss — image segmentation
Dice = 1 − (2|P∩G|) / (|P|+|G|)
P = predicted mask, G = ground truth mask

Dice coefficient = overlap between predicted and true segmentation. Loss = 1 − Dice.

Handles extreme class imbalance in segmentation (e.g. small tumour in large image).

Medical image segmentation Semantic segmentation Object detection

Decision framework — 4 questions
Q1 — What is your output type?
Continuous number → regression loss family
→ Start with MSE

Binary yes/no → binary cross-entropy
→ Pair with sigmoid output

3+ categories → categorical cross-entropy
→ Pair with softmax output

Distribution → KL divergence / ELBO
→ VAE, distillation tasks

Embedding similarity → triplet / contrastive
→ Face recognition, RAG
Q2 — Are outliers a problem?
No outliers (normal errors)
→ Use MSE — smooth gradients, assumes normality

Some outliers
→ Use Huber loss — quadratic near 0, linear for large errors. δ=1.0 is default.

Many/extreme outliers
→ Use MAE — linear penalty, completely robust

Quick check: if RMSE >> MAE (ratio > 1.5) → outliers are dominating → switch to Huber or MAE.

PySpark: RegressionEvaluator(metricName="rmse") vs metricName="mae"
Q3 — Is your data imbalanced?
Balanced classes (50/50 or close)
→ Standard binary/categorical cross-entropy

Moderate imbalance (80/20)
→ BCE with class_weight parameter
→ Spark: weightCol in LogisticRegression

Severe imbalance (<5% minority)
→ Focal loss (γ=2 default)
→ Focus on hard misclassified examples
→ Used in fraud detection, rare disease diagnosis

Extreme (<0.1%)
→ Anomaly detection + Dice loss or custom loss
Q4 — Does relative or absolute error matter?
Absolute error matters (same units as target)
→ MSE, MAE, Huber
→ E.g. predicting temperature in °C

Relative % error matters
→ MAPE or sMAPE
→ E.g. forecasting sales across products of very different scales

Ranking / similarity matters
→ Triplet loss, contrastive loss
→ RAG embedding quality, recommendation systems

Distribution matching
→ KL divergence, Wasserstein distance
→ VAEs, GANs, knowledge distillation

# F.kl_div(F.log_softmax(pred, dim=1), target_probs, reduction='batchmean') # ── Triplet loss — for embeddings ───────────────────────────── nn.TripletMarginLoss(margin=1.0) # d(anchor,pos)+margin < d(anchor,neg) # ── Contrastive loss (Siamese network) ──────────────────────── def contrastive_loss(d, y, margin=1.0): # y=1 same class, y=0 different class return (y * d**2 + (1-y) * F.relu(margin - d)**2).mean() # ── ELBO for VAE ────────────────────────────────────────────── def elbo_loss(recon_x, x, mu, logvar): recon = F.binary_cross_entropy(recon_x, x, reduction='sum') kl = -0.5 * torch.sum(1 + logvar - mu**2 - logvar.exp()) return recon + kl # ── PySpark — regression losses ─────────────────────────────── from pyspark.ml.evaluation import RegressionEvaluator for m in ["rmse","mae","r2","mse"]: print(m, RegressionEvaluator(metricName=m).evaluate(preds)) # ── PySpark — classification with class weights ─────────────── from pyspark.sql import functions as F counts = df.groupBy("label").count().toPandas() total = counts["count"].sum() weights = {int(r["label"]): total/(2*r["count"]) for _,r in counts.iterrows()} df = df.withColumn("weight", F.when(F.col("label")==1, weights[1]).otherwise(weights[0])) model = LogisticRegression(weightCol="weight", labelCol="label")
